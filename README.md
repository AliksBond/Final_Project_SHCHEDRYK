# Personal Assistant (CLI)

**Мета проекту:** Створити систему для зберігання та взаємодії з записами адресної книги та нотатками через командний рядок.

## Опис проекту

Проект реалізує персонального помічника у вигляді CLI (Command Line Interface).  
Можна додавати, переглядати та редагувати контакти та нотатки.  
Дані зберігаються на диску у файлах JSON, щоб інформація не губилася після перезапуску.

## Класи та методи

### 1. `Contact` (contacts.py)
- **Опис:** Клас для збереження інформації про контакт.
- **Атрибути:**  
  - `name` – ім’я контакту  
  - `phone` – номер телефону  
  - `email` – email адреса  
- **Методи:**  
  - `__str__()` – повертає текстове представлення контакту  

---

### 2. `Note` (notes.py)
- **Опис:** Клас для збереження нотаток.
- **Атрибути:**  
  - `title` – назва нотатки  
  - `content` – текст нотатки  
  - `created_at` – дата створення  
- **Методи:**  
  - `__str__()` – повертає текстове представлення нотатки  

---

### 3. `PersonalAssistant` (assistant.py)
- **Опис:** Основний клас, який керує контактами та нотатками.
- **Атрибути:**  
  - `contacts` – список об’єктів `Contact`  
  - `notes` – список об’єктів `Note`  
  - `storage` – об’єкт класу `Storage`  
- **Методи:**  
  - `add_contact(contact)` – додає контакт  
  - `show_contacts()` – виводить список контактів  
  - `add_note(note)` – додає нотатку  
  - `show_notes()` – виводить список нотаток  
  - `save()` – зберігає дані на диск  
  - `load()` – завантажує дані з диску  

---

### 4. `Storage` (storage.py)
- **Опис:** Відповідає за збереження та завантаження даних.
- **Методи:**  
  - `save_data(data, filename)` – зберігає словник або список у файл JSON  
  - `load_data(filename)` – читає файл JSON і повертає дані  

---

### 5. `utils.py`
- **Опис:** Допоміжні функції (наприклад, валідація введених даних користувачем)
